# Usability Test: Interactive Portraits in a Gallery Prototype

## Overview
A moderated usability session tested an interactive gallery prototype that uses a webcam to place visitors’ live portraits within historical poster imagery. The system analyzes the background image’s color palette and applies it to the user’s portrait to harmonize colors. The test explored usability, engagement, perception of historical content, and willingness to purchase a print.

## Participants
- SPK_B: Facilitator/designer demonstrating the prototype
- SPK_C: Test participant providing feedback
- SPK_A: Support (recording, notes)

## Goals
- Assess how intuitive the live portrait feature is
- Understand user desire for control over portrait aesthetics (especially color)
- Gauge how clearly background images communicate historical context
- Explore interest in taking home prints (free vs. paid)
- Capture branding/operational notes (e.g., “use the duck”)

## Prototype Summary
- Live webcam feed places the user inside a poster frame.
- Portrait colors are automatically derived from the background image’s palette.[idx-27]
- Background rotates through a database of historical posters (e.g., campus events, feminist art, political activism).
- Low-fidelity prototype; image shuffling is present but subtle.[idx-64]

## Key Findings
- Portrait feature appreciation: The participant liked the live portrait function; it increased engagement and attention to the wall and helped notice information in the images.[idx-70]
- Color control preference: The participant preferred having the option to control portrait colors directly rather than relying solely on auto-derived palettes.
- Historical messaging clarity: Without explanation, posters can be perceived as random images. The participant could glean some information once guided but found it hard to see or interpret messages at first glance.
- Image change feedback: The system shuffles images, but changes are “not very obvious.” Stronger cues would help users notice updates.
- Print interest: The participant would take a print as a free souvenir but would be unlikely to pay, indicating limited willingness to purchase.

## Usability Feedback
- Control vs. automation: Auto color harmonization is interesting but can feel like being “in control, but not in control.” Offer manual overrides for color.
- Visibility of content: Enhance legibility and context for historical messages (captions, labels, brief descriptions, or guided prompts).
- State changes: Make image transitions more noticeable (animations, visual indicators, on-screen notifications).
- Focus behavior: Color relationships between portrait and background can direct user attention effectively.

## Suggestions and Opportunities
- Add manual color picker or palette presets.
- Provide concise contextual overlays or info panels for posters (e.g., location references like Sydney University/City Road, time period, movement).
- Consider guided discovery prompts (e.g., “Find a message in this poster”) to encourage deeper engagement.
- Improve shuffle feedback (e.g., “New poster loaded” badge or subtle transition effects).
- Position prints as free souvenirs rather than paid products; explore alternative value adds (e.g., branded stamps, limited runs, or on-site sharing).

## Operational Notes
- The prototype is low fidelity but conceptually complex.[idx-64]
- Large image database present; shuffling mechanic works but needs clearer signals.
- Branding note captured: “Use the duck” (logo/branding idea); ensure this feedback is prominent in reporting.[idx-147]
- Recording and short notes were taken (“Open Records” mentioned by SPK_A).
